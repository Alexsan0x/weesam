1. PROJECT OVERVIEW
================================================================================

Dalili (Arabic for "My Guide") is a secure, intelligent tourism web application
designed to enhance visitors' experiences in Jordan. The platform provides:

    - An interactive map with 13+ tourist destinations across Jordan
    - An AI-powered virtual assistant named "Abu Mahmoud" that communicates
      in both Arabic and English
    - User accounts with the ability to save favorite places
    - A historical timeline view of Jordan's attractions
    - Responsive design that works on desktop, tablet, and mobile
    - Secure authentication with encrypted passwords

The name "Dalili" was chosen because it means "My Guide" in Arabic, which
perfectly aligns with the application's purpose of guiding tourists through
Jordan's rich cultural heritage and natural beauty.

2. PROBLEM STATEMENT AND MOTIVATION
================================================================================

Problem:
    Jordan attracts millions of tourists annually, yet there is no unified,
    smart platform that helps visitors:
    - Locate attractions easily on an interactive map
    - Access accurate cultural and historical information
    - Receive personalized, AI-driven recommendations
    - Communicate with a virtual guide in their preferred language

Motivation:
    Jordan's tourism sector is a key pillar of its economy. By combining
    modern web technologies with artificial intelligence and geospatial data,
    Dalili aims to make tourism information more accessible, engaging, and
    personalized. This supports Jordan's Vision 2025 goals for digital
    transformation in the tourism sector.

Solution:
    A web application that integrates Google Maps API for geographic
    visualization, Google Gemini AI for natural-language assistance, and
    PHP/MySQL for secure backend logic, creating a comprehensive tourism
    companion for visitors exploring Jordan.

3. TECHNOLOGY STACK
================================================================================

Frontend:
    - HTML5           Semantic page structure and content
    - CSS3            Responsive layout, animations, and visual design
    - JavaScript      Client-side interactivity and API communication
    - Font Awesome    Icon library for UI elements
    - Google Fonts    Typography (Poppins + Cairo font families)

Backend:
    - PHP 7.4+        Server-side logic, routing, and security
    - PostgreSQL      Relational database for user data and favorites
    - PDO             Secure database access with prepared statements

External APIs:
    - Google Maps JavaScript API    Interactive maps, markers, and directions
    - Google Gemini AI API          Natural language processing for the AI
                                    assistant (gemini-2.0-flash model)

Reasoning for Gemini AI Choice:
    Google Gemini was selected over other AI models because:
    1. Excellent Arabic language support (critical for Jordanian tourism)
    2. Low latency with the gemini-2.0-flash model
    3. Strong knowledge of Middle Eastern geography and culture
    4. Free tier availability for development and testing
    5. Seamless integration alongside Google Maps API
    6. Supports system instructions for character persona (Abu Mahmoud)

4. SYSTEM ARCHITECTURE
================================================================================

The application follows a standard three-tier web architecture:

    +---------------------------------------------------------+
    |                    PRESENTATION LAYER                    |
    |  HTML/CSS/JavaScript     Browser     Responsive UI      |
    +---------------------------------------------------------+
                                |
                                v
    +---------------------------------------------------------+
    |                     APPLICATION LAYER                    |
    |  PHP Pages    API Endpoints    Session Management        |
    |  config.php   api/chat.php     api/places.php            |
    |  db.php       api/auth.php     api/favorites.php         |
    +---------------------------------------------------------+
                                |
                                v
    +---------------------------------------------------------+
    |                       DATA LAYER                         |
    |  PostgreSQL Database      JSON Data Files                 |
    |  (users, favorites)       (places.json)                  |
    +---------------------------------------------------------+
                                |
                                v
    +---------------------------------------------------------+
    |                   EXTERNAL SERVICES                      |
    |  Google Maps API              Google Gemini AI API       |
    |  (Maps, Markers, Directions)  (Chat, Recommendations)   |
    +---------------------------------------------------------+

Request Flow:
    1. User opens a page in the browser
    2. PHP processes the request, includes header/footer
    3. JavaScript loads data via API endpoints (AJAX/fetch)
    4. API endpoints query the database or external services
    5. Response is rendered in the browser dynamically

5. DATABASE DESIGN
================================================================================

The MySQL database (dalili_db) contains two tables:

Table: users
    +----------------+------------------+-----------------------------------+
    | Column         | Type             | Description                       |
    +----------------+------------------+-----------------------------------+
    | id             | INT (PK, AI)     | Unique user identifier            |
    | name           | VARCHAR(100)     | User's full name                  |
    | email          | VARCHAR(255)     | User's email (unique)             |
    | password_hash  | VARCHAR(255)     | Bcrypt-hashed password            |
    | created_at     | TIMESTAMP        | Account creation timestamp        |
    +----------------+------------------+-----------------------------------+

Table: favorites
    +----------------+------------------+-----------------------------------+
    | Column         | Type             | Description                       |
    +----------------+------------------+-----------------------------------+
    | id             | INT (PK, AI)     | Unique favorite record ID         |
    | user_id        | INT (FK)         | References users.id               |
    | place_id       | VARCHAR(50)      | Place identifier from JSON data   |
    | created_at     | TIMESTAMP        | When the favorite was saved       |
    +----------------+------------------+-----------------------------------+
    Constraints: FOREIGN KEY on user_id, UNIQUE on (user_id, place_id)

Why this design:
    - Tourist place data is stored in a JSON file because it is static
      content that does not change frequently and does not require relational
      queries.
    - User data and favorites are stored in PostgreSQL because they involve
      dynamic operations (registration, login, adding/removing favorites)
      that benefit from ACID compliance and prepared statements.

8. API ENDPOINTS
================================================================================

All API endpoints return JSON responses.

GET /api/places.php
    Parameters: category (optional), search (optional)
    Returns: List of tourist places, filtered if parameters provided
    Authentication: Not required

POST /api/chat.php
    Body: JSON { "message": "user's question" }
    Returns: AI-generated response from Abu Mahmoud
    Authentication: Not required
    Note: Proxies requests to Google Gemini API

POST /api/auth.php
    Parameters: action (login|register), email, password, name (for register)
    Returns: Success/failure with message
    Authentication: Not required

GET/POST /api/favorites.php
    Parameters: action (list|add|remove), place_id (for add/remove)
    Returns: List of favorites or success/failure message
    Authentication: Required (session-based)

9. AI INTEGRATION (GEMINI API)
================================================================================

The AI assistant "Abu Mahmoud" is powered by Google's Gemini AI API
(gemini-2.0-flash model) and is integrated through a server-side proxy
pattern for security:

How it works:
    1. User types a message in the chat interface
    2. JavaScript sends the message to api/chat.php via fetch()
    3. PHP constructs a request to Gemini API with a system prompt
    4. The system prompt defines Abu Mahmoud's personality and knowledge
    5. Gemini processes the message and returns a response
    6. PHP relays the response back to the browser
    7. JavaScript displays the response in the chat window

System Prompt (defines Abu Mahmoud's behavior):
    - Acts as a friendly, knowledgeable tourism guide for Jordan
    - Knows about Petra, Wadi Rum, Dead Sea, Jerash, and other attractions
    - Provides travel tips, historical context, cultural insights
    - Responds in the same language the user writes in (Arabic or English)
    - Keeps responses concise (2-4 paragraphs)
    - Stays focused on Jordan tourism topics

Why server-side proxy:
    - API key is never exposed to the browser
    - Server can validate and sanitize user input
    - Rate limiting can be added at the PHP level
    - Response can be cached if needed

11. SECURITY MEASURES
================================================================================

The application implements several security best practices:

    1. Password Hashing
       All passwords are hashed using PHP's password_hash() with
       PASSWORD_DEFAULT (bcrypt). Plain text passwords are never stored.

    2. Prepared Statements
       All database queries use PDO prepared statements to prevent
       SQL injection attacks.

    3. Input Sanitization
       The sanitize() function uses htmlspecialchars() with ENT_QUOTES
       and UTF-8 encoding to prevent XSS (Cross-Site Scripting) attacks.

    4. Session Management
       PHP sessions are used for authentication state. Session IDs
       are regenerated on login to prevent session fixation.

    5. API Key Protection
       The Gemini API key is stored server-side in config.php and
       never exposed to the client. Chat requests are proxied
       through api/chat.php.

    6. Access Control
       The favorites page and favorites API enforce authentication
       checks before allowing access to user-specific data.

    7. CSRF Consideration
       Forms use POST method with server-side validation.

12. SETUP AND INSTALLATION GUIDE
================================================================================

Prerequisites:
    - PHP 7.4+ with pdo_pgsql extension
    - PostgreSQL 13+
    - A web browser (Chrome, Firefox, Edge, or Safari)
    - Google Maps API key (from Google Cloud Console)
    - Google Gemini API key (from Google AI Studio)

Step-by-step installation:

    1. Install PostgreSQL
       Download from https://www.postgresql.org/download and install.
       Remember the password you set for the 'postgres' user.

    2. Copy project files
       Place the project folder anywhere on your system.

    3. Create the database
       Open pgAdmin or a terminal and run:
           psql -U postgres
           CREATE DATABASE dalili_db;
           \c dalili_db
           (then run the CREATE TABLE statements from setup.sql)
       Or import setup.sql directly:
           psql -U postgres -f setup.sql

    4. Configure API keys
       Open config.php and replace:
           $google_maps_api_key = 'YOUR_GOOGLE_MAPS_API_KEY';
           $gemini_api_key = 'YOUR_GEMINI_API_KEY';
       with your actual API keys.

    5. Getting a Google Maps API Key
       - Go to https://console.cloud.google.com
       - Create a new project
       - Enable "Maps JavaScript API"
       - Go to Credentials and create an API key
       - Optionally restrict it to your domain

    6. Getting a Gemini API Key
       - Go to https://aistudio.google.com/apikey
       - Sign in with your Google account
       - Click "Create API Key"
       - Copy the key

    7. Start the PHP development server
       Open a terminal in the project folder and run:
           php -S localhost:8000

    8. Access the application
       Open your browser and go to: http://localhost:8000/
