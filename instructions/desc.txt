ABSTRACT
The Dalili Web Application is a secure and intelligent tourism platform aims to enhance visitors’ experiences in Jordan. The project seeks to address the lack of a unified, smart system that can assist tourists in easily locating attractions, accessing accurate cultural information, and receiving personalized recommendations. To solve this, the project will integrate three main technologies: Google Maps API for interactive geographic visualization, Gemini AI (Google AI) for natural-language assistance, and web security measures to ensure safe user interactions. The system will be developed using HTML, CSS, and JavaScript and will include a user-friendly interface along with a virtual assistant named “Abu Mahmoud,” which is intended to respond to users’ queries in both Arabic and English. System testing will be conducted to evaluate the platform’s ability to accurately display tourist site locations, provide relevant contextual information, and deliver intelligent responses through the AI assistant. The expected outcomes are to demonstrate the effectiveness of combining artificial intelligence with geospatial data to create an engaging and secure digital tourism experience. In conclusion, Dalili is expected to enhance accessibility, safety, and user engagement in Jordan’s tourism sector and may serve as a scalable model for other destinations seeking smart and interactive visitor platforms.

CHAPTER 2:	 LITERATURE REVIEW
2.1 	Introduction
Tourism guidance platforms are digital systems designed to provide visitors with access to locations, maps, and essential information that enhances their travel experience. These systems often combine interactive mapping tools, location data, and informational content to help users navigate destinations more easily. Existing tourism platforms vary in functionality, ranging from simple map-based applications to more advanced systems that include recommendations and automated assistance.
This chapter reviews a range of tourism-related systems and applications to understand how they operate, what services they provide, and how they support users. The goal is to identify common features, current capabilities, and existing gaps in these systems. This analysis forms the foundation for positioning the Dalili platform and clarifying how it extends beyond current solutions through smart navigation and AI-driven assistance.
2.2 	Existing Systems
Several digital tourism platforms have been developed to assist travelers by providing interactive maps, detailed location information, and relevant travel data. Among the most prominent systems is VisitJordan (Jordan, n.d.), an online platform that presents tourist attractions in Jordan through structured informational pages, images, and general descriptions. The platform enables users to explore major sites and gain insights into available activities.
Another notable system is Google Travel (Google, n.d.), which offers interactive maps, recommended destinations, and travel suggestions tailored to user preferences. This platform allows users to access points of interest, navigate locations efficiently, and obtain essential information regarding each site.
While these platforms provide valuable informational content and basic navigation support, they primarily rely on user-initiated searches and static data. They do not incorporate real-time intelligent assistance for smart guidance, which represents an opportunity for the Dalili project to introduce enhanced functionality and user experience.

2.3 	Overall Problems of Existing Systems
Although current tourism platforms offer useful information, they still have several limitations that reduce their overall effectiveness. Most of these systems rely on the user to manually search for places, which makes the experience less smooth—especially for tourists who are not familiar with the area. In many cases, the information provided is static and does not adapt to the user’s location or needs.
Another problem is the lack of real-time guidance and smart recommendations. These platforms do not inform users about the best routes, nearby attractions, or updates related to traffic or crowded areas. Many systems also lack localized content specifically tailored to the travel experience inside Jordan, making the information too general at times.
Finally, tourists often need to switch between multiple apps—one for maps, one for information, and another for recommendations. This creates a fragmented experience and slows down the process of finding the right information. These issues highlight the need for a platform like Dalili, which provides smarter guidance, stronger localization, and a more integrated user experience.

2.4 	Overall Solution Approach
The proposed solution addresses the limitations of existing tourism platforms by integrating intelligent assistance, interactive navigation, and localized content into a unified system. The Dalili Web Application combines Google Maps API for dynamic geographic visualization with Gemini AI to provide real-time, personalized recommendations and responses to user queries. Instead of relying on static information or manual searching, the system delivers an adaptive and interactive experience that guides tourists based on their preferences, location, and interests.
The solution offers a centralized platform where users can explore tourist attractions, view detailed descriptions, and access navigation tools without switching between multiple applications. The virtual assistant “Abu Mahmoud” enhances the user experience by offering culturally relevant, AI-powered guidance in Arabic and English. The platform also integrates essential security measures to ensure safe user interactions and protect API keys and data exchanges. Through this combined approach, Dalili provides a more efficient, intelligent, and user-friendly alternative to existing tourism systems, ultimately improving accessibility and engagement for visitors in Jordan.

 4.1 Introduction
The development of the Dalili Web Application followed an Agile methodology, specifically an iterative and incremental development approach. This methodology was chosen because the project involves integrating multiple dynamic components—such as Google Maps API, Gemini AI, and a responsive user interface—that require continuous refinement, frequent testing, and user feedback. The flexibility of Agile made it suitable for handling evolving requirements, especially during the integration of intelligent recommendation features and map-based navigation. Each iteration enabled the team to test system components early, identify issues, and make improvements before moving to the next phase. Industry standards and frameworks such as MVC architecture, RESTful API design principles, and Google Material Design guidelines influenced the project’s development approach, ensuring scalability, usability, and clean separation between system components.

4.2	Project Development Approach
The Dalili Web Application was developed using an Agile software development methodology, which emphasizes flexibility, iterative progress, and continuous improvement. Agile was chosen because the project includes dynamic and evolving components—such as AI-based recommendations, Google Maps integration, and responsive user interface enhancements—that require frequent testing, feedback, and refinement. Unlike linear models such as Waterfall, Agile enables the team to adapt quickly whenever modifications are needed in design, functionality, or data handling.
The Agile methodology provided several advantages for this project, including early detection of issues, the ability to refine features based on user experience, and faster integration of intelligent functionalities. It also supported parallel work among team members, allowing front-end, back-end, and AI components to be built and improved simultaneously.

4.3	System Architecture

The Dalili Web Application follows a multi-layer client–server architecture that integrates external APIs while providing a smooth and secure user experience.
Architecture Layers:
1. Presentation Layer (Front-End)
This layer contains all elements that the user interacts with. It is built using:
•	HTML for page structure
•	CSS for styling and layout
•	JavaScript for dynamic behavior and API communication
Responsibilities:
•	Displaying Google Maps with tourist locations
•	Interacting with the AI assistant “Abu Mahmoud”
•	Showing site descriptions, images, and navigation options
•	Responding to user searches and filters
•	Providing a responsive design for mobile and desktop

2. Application Logic Layer
This layer manages system functionality and processes user actions.
Responsibilities:
•	Sending requests to Google Maps API
•	Sending and receiving messages from Gemini AI API
•	Processing map markers and directions
•	Handling searches, filtering, and location retrieval
•	Ensuring security for API keys and data exchange

3. Data Layer
This layer manages the storage of tourist site data and static resources.
Includes:
•	JSON data for attractions (coordinates, descriptions, names)
•	Cloud-hosted images and assets
•	Optional caching of AI-generated content

4. External APIs (Critical Components)
Two major APIs are integrated:
a. Google Maps API
Provides interactive map
Allows markers, navigation, zooming, and real-time viewing
b. Gemini AI API
Provides smart recommendations and natural-language responses
Supports Arabic and English interaction

4.5 Testing Strategy
Testing Methodology:
The Dalili web application follows a comprehensive testing approach to ensure reliability, functionality, and user satisfaction. The methodology includes:
•	Unit Testing: Individual components, such as functions and modules, are tested to verify they perform as expected.
•	Integration Testing: Interaction between different modules and APIs (e.g., Google Maps API, Gemini AI API) is tested to ensure seamless communication and correct data flow.
•	System Testing: The entire application is tested as a whole to confirm that all features work together correctly and meet the specified requirements.
•	User Acceptance Testing (UAT): Real users test the application to provide feedback on usability, interface design, and overall experience, ensuring it meets user expectations.
Error Identification and Resolution:
Errors and issues are identified through automated testing tools, browser developer tools, and user feedback. Each identified issue is logged, prioritized, and addressed promptly. Regression testing is conducted after fixes to ensure no new issues are introduced.
Test Cases and Validation:
Test cases cover key functionalities, including map interactions, AI assistant responses, search functionality, and responsive design. For each test case:
•	Input/Action: Specific user action or input is defined.
•	Expected Outcome: The correct behaviour or response of the system is described.
•	Validation Technique: The actual output is compared with the expected outcome to confirm correctness.
